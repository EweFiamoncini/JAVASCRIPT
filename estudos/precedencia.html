Precedencia de Execução

Em JavaScript, a precedência de execução, também conhecida como "precedência de operadores", determina a ordem em que as operações são avaliadas em uma expressão. É semelhante à ordem das operações em matemática, onde a multiplicação e a divisão são calculadas antes da adição e subtração.

Dicas para Evitar Problemas

Para garantir que seu código seja claro e se comporte exatamente como você espera, a melhor prática é sempre usar parênteses ( ) para forçar a ordem de execução que você deseja. Isso não só elimina a ambiguidade, mas também torna o código mais fácil de ler para você e para outros desenvolvedores.

  • Ruim: resultado = 5 + 3 * 2;
  • Bom: resultado = 5 + (3 * 2);
  • Também bom: resultado = (5 + 3) * 2; (se essa for a ordem desejada)
